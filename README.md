<h2>Concepto de algoritmo<h2/>
<h4></h4>Para tener una estructura sólida es necesario tener en cuenta estos tres elementos<h4/>

<h2>Entradas<h2/>
<h4>Estos son los insumos para solucionar el problema lo podemos ver como imágenes audio nombre fechas […]</h4>

<h2>Procesos<h2/>
<h4>Son las acciones a ejecutar para procesar los datos que ya se tomaron en la entrada</h4>

<h2>Salidas<h2/>
<h4>Esta es la solución al problema planteado, en caso de no tener una solución podemos concluir que el algoritmo está malo </h4>

<h2>Diferencia entre usuario y programador<h2/>
<h4>Esto es tan simple como verlo que el usuario no ve el proceso detrás de la solución del problema simplemente espera un resultado o solución para el problema planteado, no tiene muy en claro que sucede con los datos brindados en la entrada

El programador por su parte tiene acceso a esta información de que se hace con los datos y todo el proceso para darle solución a este problema

Como todo debe de tener unas condiciones:

Que el algoritmo sea finito (este se debe de caracterizar para tener un fin, de no ser así el problema no estaría solucionado), que el algoritmo resuelva un problema( las instrucciones deben ser objetivas y claras y que tengan un objetivo claro), interactuar con datos externos(si se procesan los mismos datos siempre y no se pueden modificar el algoritmo de toda forma va a servir pero solo para solucionar un problema en específico y no una problemática; este resolvería una particularidad y no un tipo, este busca sistematizar la solución).</h4>

<h1>Ejemplo de Python:<h1/>

# Definimos tres variables
a, b, c = 4, 3, 2

# Realizamos una operación aritmética
d = (a + b) * c

# Mostramos el resultado
print(f"El valor de (a+b)*c es {d}")   

Este código imprime: El valor de (a+b)*c es 14.
 La sintaxis de Python es clara y se asemeja al lenguaje natural, lo que facilita su lectura y comprensión.
 Además, se puede utilizar la función print() para mostrar valores de variables, como en print(x) cuando x = 5
